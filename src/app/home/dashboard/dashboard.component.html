A dashboard usually contains items of some importance, that need to be addressed right away: notices to be sent, reports that need to be completed, newly submitted loan applications, etc.

To the left, a nav bar shows other topics that may be of interest to the user. In this sample app, I'm loading placeholder Monster and Race data from the Dungeons & Dragons 5e SRD. The data is loaded from a query to the backend.

The nav bar can be hidden and shown using the panel collapse or expand icons at the upper left, and also responds to changes in page size, overlapping the content when the width is below 551px.

Finally, context-sensitive help can be seen by clicking the ? icon to the upper right, and the user can log out by clicking the menu to the right of the ? icon.
